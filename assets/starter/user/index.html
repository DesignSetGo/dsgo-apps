User & permissions

Identify the visitor, gate UI by capability, and see what happens when an app calls a surface it didn’t declare.

Live: dsgo.user.current()

Permission: user

Returns the visitor’s record, or null when anonymous.

  // anonymous visitor

Live: permission denied

This starter’s manifest declares all 7 permissions, so a denial demo from this app would require removing one. Instead, here’s the call shape and the error you’d see in an app whose manifest is missing the relevant permission:

// In an app whose manifest does NOT include "posts" in permissions.read:
try {
  await window.dsgo.posts.list();
} catch (err) {
  console.log(err.code);    // "permission_denied"
  console.log(err.message); // "permission `posts` not granted in manifest"
}
The denial pattern Every bridge method rejects with an object shaped { code, message }. Common codes: permission_denied, not_found, invalid_input, quota_exceeded. Always wrap calls in try/catch and surface the code — users can copy it into a support request.
