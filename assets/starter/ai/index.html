AI prompt

dsgo.ai.prompt() sends a prompt through the site’s configured WordPress Connector (Anthropic, OpenAI, Google, …). The site admin pays for inference; your app never holds an API key.

Real call, real budget Clicking “Run” below makes a live LLM request that costs the site admin money (cents at most for short prompts). Don’t spam it. If no Connector is configured at Settings → Connectors in WP admin, the call will return connector_unavailable.

Live: dsgo.ai.prompt({ messages, … })

Permission: ai

No run yet.




const result = await window.dsgo.ai.prompt({
  messages: [
    { role: 'system', content: 'You are a helpful assistant.' },
    { role: 'user',   content: 'Reply with a haiku about WordPress.' },
  ],
  max_tokens: 256,        // optional
  tools: 'auto',          // optional: 'auto' | string[] of ability names
});

console.log(result.content);     // assistant's reply
console.log(result.tool_calls);  // any abilities the model invoked
console.log(result.usage);       // { input_tokens, output_tokens }


      

        How tool use works
        If your manifest declares abilities.consumes, the harness automatically exposes those abilities as tools to the model. The model can call them; the harness validates each call, executes it, and feeds the result back — all inside a single ai.prompt call. ai.max_tool_calls in the manifest caps the loop (default 5).
      


      
The full method shape


// Returns
type AiResult = {
  content: string;
  usage: { input_tokens: number; output_tokens: number };
  tool_calls: Array<{
    name: string;
    args: Record<string, unknown>;
    result:
      | { ok: true;  data: unknown }
      | { ok: false; error: string; code: string };
    duration_ms: number;
  }>;
};
