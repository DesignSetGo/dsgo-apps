Storage

Persistent JSON storage scoped to your app. Two flavors: app-scoped (shared across all visitors) and user-scoped (per-(app, user)).

Heads up These demos write real data. The shared counter below is incremented by every visitor — that’s the point of storage.app. If that’s not what you want, use storage.user instead (the second demo).

Live: dsgo.storage.app (shared counter)

App-scoped, no permission needed (always allowed). Quota: 256 KB per app per site.

Current value: …

Loading from storage…

// Read
const value = (await window.dsgo.storage.app.get('counter')) ?? 0;

// Write
await window.dsgo.storage.app.set('counter', value + 1);

Live: dsgo.storage.user (your private note)

User-scoped. Reads return null when anonymous. Writes require login — anonymous set() rejects with not_authenticated.

Loading from storage…

// Read (always allowed; returns null for anon)
const note = await window.dsgo.storage.user.get('note');

// Write (rejects with `not_authenticated` for anon)
await window.dsgo.storage.user.set('note', 'hello');
When to reach for which storage.app for things everyone should see (a featured-of-the-week, a global counter, cached config). storage.user for per-visitor preferences (theme, last-opened tab, todo lists). Both store JSON-serializable values; the bridge encodes/decodes for you.
